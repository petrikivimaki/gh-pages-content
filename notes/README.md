# Publisher Docs

Publisher turns a folder of Markdown files into a public, Obsidian-style knowledge base for GitHub Pages. This docs vault introduces the project and also acts as live sample content for testing search, file browsing, backlinks, metadata, tables, code, maps, and wiki links.

## The idea

Keep your writing as plain text, keep your publishing site as static files, and let the browser do the reading experience. Publisher can load content from a public GitHub repository, or from this local `docs` manifest while you are developing.

> This vault is intentionally small. Replace these notes with your own Markdown vault when you are ready.

## Example map
